# Build, Run, main(), and Debugging

## What is Build?
Build is the process of compiling C code into an executable output.

## What is Run?
Run means executing the compiled program.

## Build vs Run
- Build: catches syntax and compilation errors.
- Run: reveals logic and runtime issues.

## What does main() do?
The program starts from `main()`.

## What does return 0 mean?
It indicates successful program completion.

## Debugging mindset
- Set breakpoints
- Step through line by line
- Watch variable values
- Compare expected vs actual output

## Better practices
- Do not use `gets`; it is unsafe.
- Use `fgets` for text input.
- Check `scanf` return values.
- Match `printf` formats with variable types.
