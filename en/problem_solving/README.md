# Problem Solving Examples

This folder is for practicing C through small scenarios after the core syntax
is familiar. The goal is not only getting the right result; it is building habits
around input validation, readable control flow, function extraction, and useful
terminal output.

## Suggested Order

| Order | File | Focus |
| --- | --- | --- |
| 1 | [`calculator_switch_case.c`](calculator_switch_case.c) | Menu and `switch` practice |
| 2 | [`mini_atm_simulation.c`](mini_atm_simulation.c) | Loop, menu, account state |
| 3 | [`password_strength_checker.c`](password_strength_checker.c) | Character checks and condition design |
| 4 | [`expense_tracker_summary.c`](expense_tracker_summary.c) | Totals, averages, reporting |
| 5 | [`inventory_stock_alert.c`](inventory_stock_alert.c) | Stock checks and alert logic |
| 6 | [`student_stats_dashboard.c`](student_stats_dashboard.c) | Min/max/average calculations |
| 7 | [`sales_report_generator.c`](sales_report_generator.c) | Report generation and formatted output |
| 8 | [`user_role_access_simulation.c`](user_role_access_simulation.c) | Role-based access logic |

## Completion Check

Treat this folder as complete after modifying at least two examples:

- Add a new input field.
- Handle invalid input.
- Make the output easier to read.
- Extract a helper function when it improves clarity.
