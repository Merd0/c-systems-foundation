# C Learning Repository (English)

## Goal
This section organizes C learning into a clean structure. Each topic follows short theory + working code + mistake-prevention notes.

## Learning Approach
- Split topics into small parts
- Compile and run working examples
- Learn from common mistakes
- Prefer safer and more readable code
- Extend existing topics instead of duplicating them

## Topic Overview
- fundamentals: variables, basic types
- input_output: printf, scanf, fgets
- control_flow: if-else, logical checks
- operators: arithmetic, math.h, casting
- debugging: basic debugging mindset
- loops: for/while/do-while, break/continue, rand
- arrays: arrays, multidimensional arrays, character arrays
- problem_solving: example questions/solutions and mini project
- notes: build/run/main/return 0 reference notes
- notes/topic_tips.md: topic-based mini tricks

## What I learned today
I clarified the difference between build and run. I learned that `main()` is the program entry point and `return 0` signals successful completion. I also reinforced formatted output with `printf`, checked input safety with `scanf`, and understood why `fgets` is safer than `gets`.
